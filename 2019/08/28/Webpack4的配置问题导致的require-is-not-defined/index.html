Webpack4 的配置问题导致的 require is not defined

Shino161
2019-08-28
Webpack, 前端

报错信息

问题原因分析和探究

工程的 Webpack 配置中一直对于 lodash 都是不进行解析的，也就是 

module: {
  noParse: /jquery|lodash/,
  ...
}

但是在 antd-design 中是引用了 lodash 的防抖函数的。var _debounce = _interopRequireDefault(require("lodash/debounce"));
解决方法就是在 noParse 中去掉 lodash
