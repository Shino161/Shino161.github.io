useState 和 useEffect 的使用

在 React hook 里使用最频繁的应该就是这两个 hook 了，但是官网的一些简单的例子和说明
总觉得非常心智模型（w
所以这里用一些实际的例子来说明 useState 和 useEffect 的食用使用

useState 🚀

useState 其实和 this.state 并无太大的区别，都是局部状态管理的一种方式。值得注意的是：

  • 通过 setState() 是一种看起来异步的状态更新过程。实际上在更新之前在合成事件和钩子函数中没法立马拿到更新后的值，形成了所谓的“异步”。
  • setState 会进行批量更新，也就是如果 setSate 不同的值，在更新的时候也只会进行合并更新
  • 初始化状态
  • 更新状态

通过对比可以很明显的发现，setFoo 这样的用法比起 this.state 确实优雅了一些 🍷

useEffect ✨

useEffect 可以说是 React hook 里最频繁使用的一个 api。

看上去似乎有点类似 Vue2.X 的 Watch ？但是实际上还是有一些区别
使用 useEffect 先要了解以下几点官方文档里有写,虽然有些心智模型

  • useEffect 被清除的问题
    一般来说在组件多次渲染的时候，旧的 useEffect 就会被清除，新的 useEffect 就会被执行
    这就意味着每次更新就会重复触发 effect
  • useEffect 的条件执行
    useEffect 支持传入依赖来触发 effect 的执行,可以传入单个和多个依赖
  • useEffect 的执行时机
    具体的详细说明可以查询官方文档。概括来讲 useEffect 是在浏览器绘制之后延迟执行，所以不能用来处理用户视觉上需要感知到的事情（比如一些更新屏幕的同步操作）。
    但是可以用来处理一些异步的操作（比如发送一些 ajax 请求，或者调用图表库的更新视图的方法）
